## 2、输出解析器（Output Parser）

本章对应 Model I/O 中的**输出**部分：把模型的**文本输出**转成程序易用的**结构化数据**（如字符串、JSON、强类型对象）。

- **Output Parsers API**：https://reference.langchain.com/python/langchain_core/output_parsers/
- **结构化输出（Structured Output）**：https://docs.langchain.com/oss/python/langchain/models#structured-output

### 2.1 为什么需要输出解析器

大模型返回的通常是**纯文本**。而在实际应用中，我们往往需要**固定格式**的数据（如 JSON、列表、对象）以便做后续逻辑、入库或展示。**输出解析器（Output Parser）** 负责把模型的原始输出转换成所需格式，并可在解析失败时进行重试或修复，是 Model I/O 中不可或缺的一环。

![输出解析器在 Model I/O 中的位置与作用](images/11/11-2-1-1.jpeg)

---

### 2.2 什么是输出解析器

**输出解析器**是 LangChain 中位于「模型输出」和「最终数据」之间的**中间层**，主要作用包括：

- **指定格式输出**：将模型文本转换为 JSON、XML、YAML 等目标格式。
- **数据校验**：确保输出符合预期的结构和类型。
- **错误处理**：解析失败时进行重试或修正。
- **格式说明注入**：通过 **get_format_instructions()** 等方法生成「希望模型输出何种格式」的说明文本，拼入提示词，引导模型直接生成可解析的内容。

### 2.3 输出解析器分类与两大方法

#### 2.3.1 分类概览

LangChain 提供多种输出解析器，常用分类概览如下：

| 解析器                   | 输出形式 / 适用场景 | 说明                                            |
| ------------------------ | ------------------- | ----------------------------------------------- |
| **StrOutputParser**      | 纯字符串            | 从模型返回中取 content 转成 str，不做结构解析。 |
| **JsonOutputParser**     | JSON                | 将模型文本解析为 JSON（dict/list）。            |
| **PydanticOutputParser** | Pydantic 模型       | 按 Pydantic 模型定义解析，得到强类型对象。      |

#### 2.3.2 两大常用方法

- **parse(result)**：将模型返回的**原始结果**（如 AIMessage）解析成**指定格式**并返回（如 dict、str、Pydantic 对象）。
- **get_format_instructions()**：返回一段**格式说明字符串**，描述「希望模型输出成什么样子」（如 JSON 的键与类型）。把这段说明拼进 Prompt，可提高模型输出可解析的成功率。

### 2.4 常用输出解析器用法

#### 2.4.1 StrOutputParser（字符串解析器）

**是什么**：LangChain 中最简单的输出解析器，从模型返回中取出 **content** 字段并转为字符串，不做额外结构解析。

【案例源码】`案例与源码-4-LangGraph框架/05_parser/StrOutputParserDemo.py`

**示例用法**（结合 ChatPromptTemplate + init_chat_model）：

```python
"""
字符串解析器 StrOutputParser：将模型输出的 content 转为字符串。
"""
from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import ChatPromptTemplate
import os
from langchain.chat_models import init_chat_model
from loguru import logger

chat_prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个{role}，请简短回答我提出的问题"),
    ("human", "请回答：{question}")
])

prompt = chat_prompt.invoke({"role": "AI 助手", "question": "什么是 LangChain？简洁回答 100 字以内"})
logger.info(prompt)

model = init_chat_model(
    model="qwen-plus",
    model_provider="openai",
    api_key=os.getenv("aliQwen-api"),
    base_url="https://dashscope.aliyuncs.com/compatible-mode/v1"
)

result = model.invoke(prompt)
logger.info(f"模型原始输出：\n{result}")

parser = StrOutputParser()
response = parser.invoke(result)
logger.info(f"解析后的结构化结果：\n{response}")
logger.info(f"结果类型：{type(response)}")
```

#### 2.4.2 JsonOutputParser（JSON 解析器）

**是什么**：将模型的**自由文本输出**解析为**结构化 JSON**。

**两种常见用法**：

1. **在提示词中直接写明**「请以 JSON 格式返回，包含字段 xxx、yyy」。
2. **借助 JsonOutputParser.get_format_instructions()**：用该方法得到一段格式说明，拼进 Prompt，让模型按该格式输出，再用 `JsonOutputParser().parse(...)` 解析。

【案例源码】`案例与源码-4-LangGraph框架/05_parser/JsonOutputParserDemo.py`（手写提示词要求返回 JSON）、`案例与源码-4-LangGraph框架/05_parser/JsonOutputParser_GetFormatInstructions.py`（使用 get_format_instructions 拼进提示词）。

### 2.5 输出解析器进阶：结构化输出（Pro+）

当希望模型返回**固定格式**的数据（例如「一定包含 name、age 两个字段的字典」），而不仅是随便一段话时，可以用「结构化输出」：先定义好「长什么样」，再让 LangChain 按这个形状去引导模型输出并解析。定义结构时，常会用到 **TypedDict**、**Pydantic** 和 **Annotated**，下面先分清这三者是什么、从哪来、能干什么。

- **官方文档**：https://docs.langchain.com/oss/python/langchain/models#structured-output

#### 2.5.0 先搞清这几个词：TypedDict、Pydantic、Annotated

| 名字 | 从哪来 | 通俗理解 | 会不会在「运行时」做校验？ |
|------|--------|----------|----------------------------|
| **TypedDict** | Python **标准库** `typing`（3.8+） | 用来**描述**一个字典「有哪些键、每个键是什么类型」。相当于给 dict 写一张「结构说明书」，给人或工具（如 LangChain）看，**不会**在运行时报错说「你多写了一个键」或「值类型不对」。 | 否，仅类型提示/说明。 |
| **Pydantic** | **第三方库**，需 `pip install pydantic` | 用来定义「带校验的数据模型」：你写一个类（继承 BaseModel），声明字段和类型；**创建对象时**会真的检查类型、长度、范围等，不合格就抛 `ValidationError`。 | 是，会按 Field、validator 等校验。 |
| **Annotated** | Python **标准库** `typing`（3.9+） | 用来在类型上**挂一段额外信息**（元数据），例如「这个 int 表示年龄，范围 0–150」。这段信息可以给 LangChain 用来生成提示词里的字段说明；在**纯 Python** 里不会自动变成「校验规则」。只有和 **Pydantic 的 Field** 一起用时，才会变成真正的范围校验。 | 单独用：否。和 Pydantic Field 一起用：是。 |

**一句话记**：**TypedDict** 和 **Annotated** 是标准库里的「说明/注释」；**Pydantic** 是第三方库，负责「真刀真枪」的校验和强类型。做结构化输出时，可以用 TypedDict 或 Pydantic 定义「长什么样」，LangChain 再根据这个定义去让模型按格式输出并解析。

---

#### 2.5.1 TypedDict：用标准库描述「字典长什么样」

**TypedDict** 来自 Python 自带的 `typing`，用来声明「一个字典里有哪些键、分别是什么类型」。它**不负责**在运行时检查你传进去的 dict 是否真的符合，主要作用是：

- 给人、IDE、LangChain 看：模型输出的 JSON 应该长什么样；
- LangChain 可以根据这个「形状」生成提示词里的格式说明，并把模型返回的 JSON 解析成 Python 的 dict。

**结合案例看**（`StructuredOutput_TypedDict.py`）：先定义「一个动物」和「动物列表」的结构，再用 `llm.with_structured_output(AnimalList)` 让模型按这个结构返回并自动解析。

```python
from typing import TypedDict, Annotated

# 定义一个「动物」的结构：两个字段，分别是动物名、表情（emoji）
class Animal(TypedDict):
    animal: Annotated[str, "动物"]   # Annotated 里写的是「描述」，给模型/LangChain 看
    emoji: Annotated[str, "表情"]

# 定义「动物列表」：一个字段 animals，类型是 Animal 的列表
class AnimalList(TypedDict):
    animals: Annotated[list[Animal], "动物与表情列表"]

# 绑定到模型后，invoke 一次就能得到符合 AnimalList 的 dict
llm_with_structured_output = llm.with_structured_output(AnimalList)
resp = llm_with_structured_output.invoke(messages)  # 例如 {"animals": [{"animal": "猫", "emoji": "🐱"}, ...]}
```

**典型步骤小结**：① 用 `class XXX(TypedDict):` 声明字段；② 可选地用 `Annotated[类型, "描述"]` 给字段加说明；③ `llm.with_structured_output(XXX)` 得到「按 XXX 输出并解析」的可调用对象；④ `.invoke(messages)` 得到解析好的 dict。

#### 2.5.2 Annotated 与 Pydantic：描述 vs 校验

**Annotated** 是在类型上「加一段说明」，例如 `Annotated[int, "年龄，范围0-150"]`。在**纯 Python + TypedDict** 里，这段说明**不会**在运行时帮你检查「年龄是不是真的在 0–150」——Python 只会关心「是不是 int」。所以你可以写 `age2=188` 而不会报错。案例见 `AnnotatedTypedDict.py`：

```python
from typing import Annotated, TypedDict

Age = Annotated[int, "年龄，范围0-150"]  # 只是「说明」，不校验

class Person(TypedDict):
    name: str
    age: int
    age2: Age

p = Person(name="z3", age=111, age2=188)  # 188 超出 0–150，但不会报错
```

**Pydantic** 则会在**创建对象时**做校验。若用 Pydantic 的 `Field(ge=0, le=150)` 和 Annotated 结合，例如 `Annotated[int, Field(ge=0, le=150, description="年龄")]`，那么赋值 188 就会抛 `ValidationError`。案例见 `AnnotatedPydantic.py`：

```python
from typing import Annotated
from pydantic import BaseModel, Field, ValidationError

Age = Annotated[int, Field(ge=0, le=150, description="年龄，范围0-150")]  # 这里会真校验

class Person(BaseModel):
    name: str
    age: int
    age2: Age

# Person(name="z3", age=11, age2=188) 会触发 ValidationError，因为 188 不在 0–150
```

**Pydantic 做结构化输出**：用 Pydantic 定义模型（如 Product：name、category、description），再用 `PydanticOutputParser(pydantic_object=Product)` 和 `get_format_instructions()` 把格式说明拼进 Prompt，模型返回后解析成 Product 实例，并可做长度、范围等校验。完整流程见 `StructuredOutput_Pydantic.py`。

**使用 Annotated 增强类型说明**：在 TypedDict 或 Pydantic 的字段里用 `Annotated[类型, "描述"]` 或 `Annotated[类型, Field(...)]`，可以让 LangChain 生成更清晰的格式说明，模型更容易按字段含义输出。

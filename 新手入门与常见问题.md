# 新手入门与常见问题

本文档面向**零基础或刚接触本项目的同学**，说明如何把项目跑起来、如何运行 Python 案例、如何申请各平台 API Key，以及常见报错怎么处理。

---

## 1、项目是什么？我能做什么？

本仓库是 **《AI 智能体实战速成指南：从零到企业级落地》** 的笔记与案例代码，包含：

- **教程笔记**：大模型基础、Coze/Dify、LangChain/LangGraph、RAG/Agent 等
- **可运行案例**：主要在 `案例与源码-4-LangGraph框架/` 下，每个 `.py` 文件都是一个独立小示例

你可以在本地**按章节学习**，并**亲手运行**每个 Python 案例，调用大模型完成对话、解析、流式输出等。完整章节结构见 [教程目录大纲](教程目录大纲.md)，案例与源码索引见 [教程案例链接汇总](教程案例链接汇总.md)。

## 2、环境准备（第一次必做）

### 2.1 需要安装什么？

- **Python**：建议 **Python 3.10 或 3.11**（3.9 也可，部分依赖可能需注意版本）
- **Git**：用于克隆或更新仓库（若已下载 ZIP 可略过）

在终端输入以下命令可查看本机 Python 版本：

```bash
python3 --version
```

若未安装 Python，请到 [Python 官网](https://www.python.org/downloads/) 下载安装，安装时勾选「Add Python to PATH」。

### 2.2 建议使用虚拟环境（避免和本机其他项目冲突）

在**项目根目录**（即能看到 `requirements.txt`、`README.md` 的那一层）打开终端，执行：

```bash
# 创建虚拟环境（会多出一个 .venv 文件夹）
python3 -m venv .venv

# 激活虚拟环境
# Windows（CMD）：
.venv\Scripts\activate
# Windows（PowerShell）：
.venv\Scripts\Activate.ps1
# macOS / Linux：
source .venv/bin/activate
```

激活成功后，命令行前面会多出 `(.venv)`。

### 2.3 安装项目依赖

仍在**项目根目录**且虚拟环境已激活时执行：

```bash
pip install -r requirements.txt
```

等待安装完成。若网络较慢，可尝试使用国内镜像，例如：

```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

## 3、如何运行 Python 案例？

### 3.1 重要：在「项目根目录」下运行

本项目的 Python 脚本会从**当前工作目录**读取根目录下的 `.env` 文件。因此需要在**项目根目录**执行 `python`，而不是在案例所在子目录里执行。

**正确示例**（在项目根目录）：

```bash
# 确保当前在项目根目录（能看到 requirements.txt、.env 等）
python 案例与源码-4-LangGraph框架/01-helloworld/StandardDesc.py
```

或先进入项目根目录再运行：

```bash
cd /path/to/ai-agents-from-zero
python 案例与源码-4-LangGraph框架/01-helloworld/LangChainV1.0.py
```

**错误示例**（在子目录里直接运行，容易导致找不到 `.env`）：

```bash
cd 案例与源码-4-LangGraph框架/01-helloworld
python StandardDesc.py   # 可能报错：找不到 .env 或 API Key 为空
```

若你希望在子目录里直接运行，需要保证该目录或其父目录下有 `.env`，或自行在代码里指定 `.env` 路径（本仓库案例默认从根目录加载）。

## 4、配置 API Key（.env 文件）

### 4.1 从示例复制并修改

1. 在项目根目录找到 **`.env-example`** 。
2. 复制一份并重命名为 **`.env`** 。
3. 用记事本或 VS Code 打开`.env` ，把里面的占位 Key（`sk-xxx...`）改成你在各平台申请的真实 API Key。
4. 保存后，**不要**把 `.env` 提交到 Git（本仓库已通过 `.gitignore` 忽略），以免泄露密钥。各平台 API Key 的获取与「调用三件套」说明见 [第 10 章 - LangChain 快速上手与 HelloWorld](10-LangChain快速上手与HelloWorld.md) 的「2、大模型服务平台」与「4、实战：基于阿里百炼的 HelloWorld」。

### 4.2 .env 里变量名要和代码一致

本仓库案例里用到的变量名包括：

- `QWEN_API_KEY`：通义千问（部分 helloworld 示例）
- `aliQwen-api`：阿里云百炼 / 通义千问（多数 prompt、parser 示例）
- `deepseek-api`：DeepSeek 接口（02-models_io 中 DeepSeek 示例）

变量名必须与上面一致（包括大小写和中间的 `-`），否则 `os.getenv("xxx")` 会取到空值，调用时报错。

### 4.3 格式示例

`.env` 内容示例（键与值之间可以有空格，值用单引号或双引号均可）：

```env
QWEN_API_KEY = 'sk-你的通义千问Key'
aliQwen-api = 'sk-你的阿里云百炼Key'
deepseek-api = 'sk-你的DeepSeek的Key'
```

## 5、各 API 平台如何申请 Key？

下面按「平台」说明如何拿到 API Key，用于填充 `.env`。

### 5.1 通义千问 / 阿里云百炼（QWEN / aliQwen）

- **用途**：本仓库大量案例使用 `aliQwen-api` 或 `QWEN_API_KEY` 调用通义千问。
- **申请步骤**：教程中有图文说明，可直接查看 [第 10 章 - LangChain 快速上手与 HelloWorld](10-LangChain快速上手与HelloWorld.md) 的「**2、大模型服务平台**」（含阿里云百炼平台、API-Key 管理入口表格）与「**4、实战：基于阿里百炼的 HelloWorld**」（含 API Key 获取、模型名、Base URL 三件套）。简要步骤如下：
  1. 打开 [阿里云官网](https://www.aliyun.com/) 并登录，若没有账号需先注册。
  2. 搜索「**百炼**」或进入「**灵积模型服务**」相关产品页（具体名称以阿里云当前产品为准）。
  3. 开通「模型服务」或「百炼」后，在控制台里找到 **API-KEY** 管理，创建或复制 Key。
  4. Key 一般为 `sk-` 开头的长字符串，复制到 `.env` 的 `aliQwen-api` 或 `QWEN_API_KEY` 中。
- **计费**：按调用量计费，新用户常有免费额度，具体以阿里云/百炼页面说明为准。

### 5.2 DeepSeek

- **用途**：本仓库中 `02-models_io` 下的 DeepSeek 示例、以及部分使用 `deepseek-api` 的脚本。
- **申请步骤**：教程中有截图与步骤，见 [第 10 章 - LangChain 快速上手与 HelloWorld](10-LangChain快速上手与HelloWorld.md)（含「多模型共存」中 DeepSeek 的 API Key 获取与配置）。简要步骤如下：
  1. 打开 [DeepSeek 开放平台](https://platform.deepseek.com/)（或搜索「DeepSeek API」）。
  2. 注册/登录后，在控制台或「API Key」页面创建 Key。
  3. 将 Key（一般为 `sk-` 开头）填入 `.env` 的 `deepseek-api`。
- **计费**：按使用量计费，官网有价格与免费额度说明。代码接入方式见 [第 11 章 - Model I/O 与模型接入](11-Model-I-O与模型接入.md) 的「3.2 接入 DeepSeek」。

### 5.3 OpenAI 兼容接口（如用 OpenAI、或其它兼容厂商）

- 若案例中使用的是 `openai` 库 + `base_url` 方式（例如接 DeepSeek、国内兼容 OpenAI 的厂商），通常只需在 `.env` 里配置对应厂商的 Key 和文档中的 `base_url` 即可。**接入方式与参数说明**见 [第 11 章 - Model I/O 与模型接入](11-Model-I-O与模型接入.md) 的「3.1 使用 OpenAI 兼容接口」与「3.3 接入通义千问（阿里云百炼）」。
- **OpenAI 官方**：需在 [OpenAI 平台](https://platform.openai.com/) 注册并创建 API Key，且需能访问其服务（部分地区可能需代理）。

### 5.4 不想用云 API：用本地 Ollama（无需 Key）

- 案例在 **`案例与源码-4-LangGraph框架/03-ollama/`** 下，使用 [Ollama](https://ollama.com/) 在本地运行模型，**不需要任何 API Key**。
- 安装、配置与 LangChain 调用方式见 [第 12 章 - Ollama 本地部署与调用](12-Ollama本地部署与调用.md)（含下载、安装、常用命令、模型拉取及 `ChatOllama` 用法）。需要你先在本机安装并启动 Ollama，拉取模型（如 `ollama run qwen:4b`），再运行该目录下的 Python 脚本；各脚本内也有注释说明。

## 6、常见问题与解决

### 6.1 报错：`ModuleNotFoundError: No module named 'xxx'`

- **原因**：依赖未安装，或不在正确的 Python 环境中运行。
- **处理**：
  1. 确认已在**项目根目录**执行过：`pip install -r requirements.txt`。
  2. 若使用了虚拟环境，确认已**激活**（命令行前有 `(.venv)`），再用 `python` 运行案例。
  3. 若某案例单独要求额外包（如 `dashscope`、`langchain-deepseek`），脚本注释里通常会写，按提示 `pip install 包名` 即可。

### 6.2 报错：API Key 无效、401、403 或提示未配置

- **原因**：`.env` 未配置、未在项目根目录运行导致读不到 `.env`、或变量名与代码不一致。
- **处理**：
  1. 确认项目根目录下有 `.env`（不是只改 `.env-example`）。
  2. 确认在**项目根目录**下执行 `python 案例与源码-4-LangGraph框架/.../xxx.py`。
  3. 检查 `.env` 里变量名是否与脚本中 `os.getenv("...")` 一致（如 `aliQwen-api`、`deepseek-api`）。
  4. 确认 Key 未多余空格、未缺字符，且对应平台控制台中该 Key 仍有效、未删除。

### 6.3 运行时报编码错误（如 `UnicodeDecodeError`）

- **原因**：`.env` 或脚本文件编码不是 UTF-8，或系统默认编码与文件不一致。
- **处理**：
  1. 用编辑器将 `.env` 和报错的 `.py` 文件保存为 **UTF-8** 编码。
  2. 本仓库案例中已使用 `load_dotenv(encoding='utf-8')`，若仍报错，可检查是否有其他文件或路径包含非 UTF-8 字符。

### 6.4 找不到 `.env` 或 `getenv` 得到空值

- **原因**：当前工作目录不是项目根目录，`load_dotenv()` 默认只在当前目录找 `.env`。
- **处理**：始终在**项目根目录**执行 Python，例如：
  `python 案例与源码-4-LangGraph框架/01-helloworld/StandardDesc.py`。

## 7、代码格式（Black）与保存时自动格式化

本仓库使用 [Black](https://github.com/psf/black) 统一 `.py` 代码风格，便于协作与阅读。**不运行案例也可忽略本节**；若你参与改代码或希望保存时自动格式化，可按下面操作。

### 7.1 命令行格式化

- **安装**：`pip install -r requirements.txt` 已包含 `black`；若仅需格式工具可 `pip install black`。
- **格式化**：在**项目根目录**执行
  - `black .` — 格式化所有 Python 文件
  - `black --check .` — 仅检查是否符合 Black 规范，不修改文件
- **排除**：Black 已通过根目录 `pyproject.toml` 排除 `.venv`、`venv`、`__pycache__` 等目录。

### 7.2 保存时自动格式化（Cursor / VS Code）

项目已配置保存时对 Python 文件自动格式化（见 `.vscode/settings.json`）。

- **使用 Black 格式化**：在 Cursor/VS Code 扩展市场搜索 **Black Formatter**（扩展 ID：`ms-python.black-formatter`，Microsoft 出品），安装后即可在保存 `.py` 文件时自动按 Black 风格格式化。
- 若未安装 Black Formatter，保存时仍会使用编辑器默认的 Python 格式化方式；若要统一为 Black，请安装该扩展并确保已执行 `pip install black`（或通过 `requirements.txt` 安装）。
